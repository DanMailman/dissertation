\chapter{Case Study and Representative Application}\label{ch:case-study}

To facilitate easily-learned, gesture-efficient, on-demand non-QWERTY input,
\mulex input applications should enable:

\begin{itemize}
	\item Easy switching between QWERTY and non-QWERTY.
	\item ``Touch typing'' non-QWERTY with high gestural efficiency.
	\item Accessibility via keyboards, mice, and other input peripherals.
	\item QWERTY access to non-QWERTY characters.
\end{itemize}

In the most general statement, and because the keyboard is among the most ubiquitous input peripherals,
the non-QWERTY input problem reduces to presenting \uline{QWERTY-accessible} choices that result in the
highest number of non-QWERTY characters for the fewest gestures. The general strategy for optimizing the
solution is to present choice sets of multi-character non-QWERTY lexemes---rather than single characters---for selection.

To initiate discussion, consider the example of laptop users in the field of linguistics writing papers in English
about aspects of Chinese. For this example, most user time and effort are spent on word processor GUIs occupying
monitors (Figure~\ref{fig:muciyu-supplementing-word}, left), and most of the document text is English, input using
laptop keyboards. Periodically, and for relatively short periods of time, users input Chinese instead of English.
For this use case, the non-QWERTY input set is Mandarin characters and lexemes.

Three resources are used to implement the \mulex strategy:
\begin{itemize}
	\item A set of Chinese characters, optionally with frequencies.
	\item A Chinese lexeme frequency list.
	\item A method for mapping QWERTY characters to Chinese characters and lexemes.
\end{itemize}

\section{Chinese character frequency lists}\label{sec:chinese-character-frequency-lists}

Given the productive nature of language, no Chinese lexicon includes every possible lexeme.
To enter lexemes not in the source frequency list, \muciyu enables input of individual \hanzi
(hànzì, ``Chinese characters''). The character frequency lists used in this research are from
\cite{Da2004ChineseCharBigramFrequencies}. For completeness, an additional list without frequencies that
includes a target set of Unicode characters with a pīnyīn pronunciation is generated programmatically.

To indicate available non-lexeme \hanzi, \muciyu suffixes the green corner display text with a `+'.

For this research, the Unicode standard is used as the resource for the complete set of Mandarin characters
\parencite{UnicodeConsortium2025Unicode170}.

\section{Chinese lexeme frequency lists from internet corpora}\label{sec:chinese-lexeme-frequency-lists}

Lexeme frequency lists can be derived from corpora using readily available tools
\parencite{BaroniKilgarriff2006LargeWebCorpora}. The two Mandarin frequency lists used for this research are:
\begin{itemize}
	\item \href{http://corpus.leeds.ac.uk/frqc/lcmc.num}{Lancaster Corpus of Mandarin Chinese} (\textbf{LCMC},
	\textasciitilde5{,}000 lexemes) \parencite{McEneryXiao2004LCMC}.
	\item \href{http://corpus.leeds.ac.uk/frqc/internet-zh.num}{Chinese Internet Corpus} (\textbf{CIC},
	\textasciitilde50{,}000 lexemes) \parencite{Sharoff2006AutomatedSearchEngineCorpora}.
\end{itemize}


\section{Pīnyīn maps QWERTY to Chinese pronunciation}\label{sec:pinyin-maps-qwerty}

The most widely used system for representing Chinese pronunciation is \textbf{\emph{pīnyīn}}; it writes Chinese using the Latin alphabet and is used throughout the world \parencite{LoC1997PinyinPressRelease}. Because the Latin alphabet used in pīnyīn consists of the same 26 letters used for English \parencite{CIA2007ProgressOfPinyin}, all non-tone pīnyīn letters are available on the QWERTY keyboard (Table~\ref{tab:pinyin-key-ipa-desc}).

Pīnyīn spelling is syllable-based. In Standard Mandarin, each syllable can be analyzed as an \emph{initial} (possibly null) followed by a \emph{final}. The initial is the onset consonant (or zero onset), and the final comprises the vocalic nucleus and any offglide and/or nasal coda. For reference, the standard set of initials is:

\begin{itemize}
	\item \textbf{Initials:} b, p, m, f, d, t, n, l, g, k, h, j, q, x, zh, ch, sh, r, z, c, s (plus null initial)
\end{itemize}

The commonly used finals include:

\begin{itemize}
	\item \textbf{Finals:} a, o, e, i, u, ü; ai, ei, ao, ou; an, en, ang, eng, ong; ia, ie, iao, iu, ian, in, iang, ing, iong; ua, uo, uai, ui, uan, un, uang; üe, üan, ün; er
\end{itemize}

Because standard pīnyīn does not use the letter \textbf{v}, many QWERTY-based pīnyīn input methods repurpose \textbf{v} to represent \textbf{ü} (and related finals such as \textbf{üe}, \textbf{üan}, \textbf{ün}). For example, \textit{nǚ} is often typed as \textit{nv}, and \textit{lüè} as \textit{lve}.

Figure~\ref{fig:ms-pinyin-steps} shows a concrete example of the interaction required to enter the word 中国 ``China'' using the Microsoft Pinyin IME in Notepad++. Starting with an English keyboard layout, the user first presses \texttt{Ctrl+Space} to activate Microsoft Pinyin (step~1). They then type the letters \texttt{z}, \texttt{h}, \texttt{o}, \texttt{n}, \texttt{g}, \texttt{g} in sequence (steps~2--7). After each letter, the composition string and candidate bar update to reflect the IME's current hypothesis about the intended syllable. On the final keystroke, \texttt{1}, the user selects candidate~1, 中国, from the candidate list and the IME commits it to the text buffer (step~8). In this example, a single lexical item requires eight keystrokes (G/Lex = 8/1) under a widely used mainstream IME.

Modern Chinese IMEs such as Microsoft Pinyin maintain an internal language model and continuously adapt the ordering of the candidate list based on corpus frequency, local context, and the user's own selection history. In principle, this maximizes the probability that the desired character or word appears in the first candidate position, thereby reducing average selection cost for short, isolated interactions. However, the same adaptivity makes it impossible for users to rely on a stable mapping from gesture sequence to lexical outcome. A given pinyin string may produce one ordering of candidates on Monday and a different ordering on Friday, or even vary within a session as the IME updates its statistics. From the perspective of touch-typing or chordal input, where users invest effort into building reliable motor programs, this instability is a fundamental obstacle: the movement pattern ``type pinyin, then press 1'' cannot be learned as a robust shorthand for a particular lexeme if the identity of the first candidate is itself a moving target. In the analysis that follows, I treat such adaptive candidate reordering as a design choice that optimizes for short-term prediction accuracy at the expense of long-term gestural consistency and muscle-memory formation.

\section{Enabling and disabling \muciyu{} with long-press}\label{sec:muciyu-enable-disable-longpress}

For the example use case, an always-visible \muciyu{} grid is unnecessary and potentially distracting. \muciyu{} therefore supports easy \textbf{\uline{enable}} and \textbf{\uline{disable}} behavior (Figures~\ref{fig:muciyu-initial-display} and \ref{fig:muciyu-supplementing-word}, red ellipse).

If \muciyu{} is not yet running (e.g., first use since boot), users \textbf{\uline{activate}} it via an OS menu selection. If \muciyu{} is already running, a \textbf{\uline{long-press}} of the \texttt{/} key \textbf{\uline{toggles it on}} (\textbf{show}) and \textbf{\uline{toggles it off}} (\textbf{hide}). \textbf{\uline{Deactivation}} (fully exiting the application) is performed via mouse-clickable menu selections accessed through the blue corner cell. Because activation is via the OS, show/hide is via long-press, and deactivation is via \muciyu{}'s own UI, none of these gestures interfere with the GFAs of concurrently running applications.

\section{Using \muciyu{} for selecting lexemes}\label{sec:muciyu-selecting-lexemes}

\subsubsection{Select the most frequent lexemes with one gesture}\label{sec:muciyu-one-gesture}

The initial \muciyu{} display shows the most frequently used lexemes according to the source corpus. When the grid is visible, users select lexemes in one of two ways. They can click the cells directly or \textbf{\uline{long-press}} the corresponding QWERTY keys indicated in blue (Figure~\ref{fig:muciyu-initial-display}, red circle). For example, tapping \texttt{K} in the usual manner inputs \texttt{k}, while long-pressing \texttt{K} inputs the lexeme \hanzi{}~这 (pīnyīn \emph{zhè}). Because selection uses long-press, \muciyu{} does not interfere with existing word-processing GFAs. Visual feedback (Figure~\ref{fig:muciyu-initial-display}, yellow highlight) indicates when the long-press threshold has been reached.

As another example, to input \hanzi{}~的 (pīnyīn \emph{de}), users long-press \texttt{A} until the cell highlights; the lexeme is inserted upon release.

The GE for any single-character lexeme on the initial \muciyu{} grid is 1.00, and the GE for \hanzi{}~一个 (\emph{yī gè}) is 2.00. By comparison, in the Microsoft Pinyin IME the number of gestures to input \hanzi{}~的 varies from 2 (\texttt{d} + \texttt{Enter}) to 3 (\texttt{d} + \texttt{e} + \texttt{1}), with corresponding GEs of 0.50 and 0.33 (not including the gestures required to switch between English and Chinese input). The lexemes on the initial display account for 18.69\% of written Chinese according to LCMC.

\subsubsection{Select the next most frequent lexemes with two gestures}\label{sec:muciyu-two-gestures}

To input lexemes not on the initial display, users \textbf{\uline{Shift+long-press}} the first letter of the lexeme’s pīnyīn to update the grid to that initial. For example, to input \hanzi{}~时候 (\emph{shí hòu}, “when”), users:
\begin{itemize}
	\item \textbf{\uline{Shift+long-press}} \texttt{S} to update the display to the \texttt{s-} grid (see Figure~\ref{fig:muciyu-s-grid}). % TODO: confirm figure label
	\item long-press \texttt{P} to insert the lexeme.
\end{itemize}

	\includegraphics[width=0.75\linewidth]{muciyu_s_grid.png}
	\caption{\muciyu{} grid of frequently used lexemes with pīnyīn beginning with \texttt{s-}}

There are 658 frequently used lexemes that can be selected with two gestures in this manner. The average GE for two-gesture lexemes is 0.80, and two-gesture lexemes account for 25.78\% of written Chinese. Combined with the one-gesture lexemes, 44.47\% of written Chinese can be input with two or fewer gestures.
